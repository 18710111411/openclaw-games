## 🧠 知识库

### 设计原则
1. **核心循环优先** - 先做最小可玩版本
2. **快速迭代** - 每周一个可测试版本
3. **玩家反馈** - 尽早测试，收集反馈
4. **成本控制** - 能用免费资源就不用付费

### 技术债务管理
- 原型阶段：快速实现，不追求完美
- 开发阶段：重构核心代码
- 打磨阶段：优化性能，修复 Bug
