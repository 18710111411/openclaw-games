# 系统重启记录

## 2026-03-14 - 第二次重启

**原因：** AI 响应越来越慢，最终像死机一样
**影响：** 项目中断，记忆丢失

### 可能的根本原因

1. [ ] 网关进程内存泄漏
2. [ ] 模型 API 限流
3. [ ] 会话上下文过大
4. [ ] 磁盘 I/O 瓶颈
5. [ ] 其他进程资源抢占

### 监控检查点

```bash
# 网关状态
openclaw gateway status

# 会话 token 使用
openclaw status | grep -i context

# 系统资源
free -h && df -h /

# 网关日志
openclaw logs --follow
```

### 预警阈值

| 指标 | 警告 | 危险 |
|------|------|------|
| Token 使用 | >60% | >80% |
| 内存可用 | <500MB | <200MB |
| 磁盘剩余 | <10GB | <5GB |
| 响应时间 | >5s | >15s |

### 应对策略

**响应变慢时：**
1. 先检查 `session_status` - 看 token 使用
2. 检查 `openclaw gateway status`
3. 如果网关异常 → `openclaw gateway restart` (不重启整个系统)
4. 如果 token 过高 → 清理会话历史

**避免完全重启：**
- 优先重启网关而非整个 OpenClaw
- 定期清理记忆文件
- 监控资源使用
