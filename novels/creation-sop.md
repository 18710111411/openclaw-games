# 创作 SOP（标准作业流程）

> 每章创作的标准流程

---

## 📋 流程总览

```
准备 → 大纲 → 创作 → 自检 → 修改 → 发布
 ↓      ↓      ↓      ↓      ↓      ↓
5min   10min  60min   10min  15min  5min
```

**总耗时：** 约 120 分钟/章
**目标产出：** 5000 字/章（4500-5500 字浮动）

---

## 1️⃣ 准备阶段（5 分钟）

### 环境准备

- [ ] 关闭无关网页/应用
- [ ] 准备好参考资料（大纲、角色设定等）
- [ ] 确保网络畅通（需要搜索时）
- [ ] 准备好水和零食（肝帝必备）

### 状态准备

- [ ] 深呼吸，进入创作状态
- [ ] 回顾上一章内容（5 分钟快速阅读）
- [ ] 明确本章目标

### 工具准备

- [ ] 打开文档编辑器
- [ ] 打开 `outline.md`（大纲）
- [ ] 打开 `characters.md`（角色设定）
- [ ] 打开 `foreshadowing-tracker.md`（伏笔追踪）
- [ ] 打开 `chapter-checklist.md`（检查清单）

---

## 2️⃣ 大纲阶段（10 分钟）

### 本章目标

根据总大纲，确定本章具体内容：

```
章节号：第____章
章节名：（暂定）
核心事件：_______________________
出场角色：_______________________
需要铺设的伏笔：_________________
需要回收的伏笔：_________________
本章爽点：_______________________
```

### 章节结构

```
开头（800 字）：
  - 场景描写（细腻）
  - 引入冲突/事件
  
发展（2500 字）：
  - 事件发展（节奏舒缓）
  - 角色互动（对话自然）
  - 冲突升级（不急于爆发）
  
高潮（1200 字）：
  - 冲突展现（不一定是爆发）
  - 爽点展现
  - 伏笔铺设（不急于回收）
  
结尾（500 字）：
  - 事件收尾
  - 新的悬念
  - 引导下一章
```

**节奏控制：**
- 不急于推进主线
- 伏笔可以长期铺设
- 困难/问题不用立刻解决
- 注重日常细节和角色互动

### 关键点记录

记录本章需要注意的点：
```
（例如：主角境界提升、某角色首次出场、某伏笔铺设等）
```

## 3️⃣ 创作阶段（60 分钟）

### 3.1 开头（10 分钟）

**目标：** 500 字，吸引读者

**要点：**
- [ ] 场景描写（视觉/听觉/嗅觉）
- [ ] 引入冲突或事件
- [ ] 设置钩子（让读者想看下去）

**示例：**
```
青云宗，外门任务堂。

李执事摩挲着下巴，看着眼前悬浮的任务面板，
嘴角露出一丝不易察觉的微笑。

"这个任务，应该会有很多人抢着接吧..."

他手指轻点，发布了穿越后的第一个任务。
与此同时，远在万里之外的玩家论坛，
一个名为"我是打工王"的玩家，
突然收到了一条系统推送...
```

---

### 3.2 发展（25 分钟）

**目标：** 2000 字，推进剧情

**要点：**
- [ ] 事件逐步发展
- [ ] 角色互动（对话 + 行动）
- [ ] 冲突逐步升级
- [ ] 适当描写细节（五感）
- [ ] 铺设伏笔（如需要）

**技巧：**
- 多用对话推进剧情（不枯燥）
- 穿插心理描写（展现主角想法）
- 适当插入背景说明（世界观）
- 保持节奏（不拖沓）

---

### 3.3 高潮（20 分钟）

**目标：** 1500 字，展现爽点

**要点：**
- [ ] 冲突爆发
- [ ] 爽点充分展现
- [ ] 伏笔回收（如需要）
- [ ] 情绪推向高点

**常见爽点写法：**

**信息差爽点：**
```
玩家："这个任务怎么这么难？"
主角：（微笑）"因为这是隐藏任务啊"
玩家："？？？你怎么知道？"
主角："我说的"
```

**身份差爽点：**
```
玩家："李执事真是个好人，任务奖励这么高"
主角：（内心）"你拿 1 万，我拿 2 万，我当然好人"
玩家："我要给李执事当狗！"
主角："准了"
```

**双倍返利爽点：**
```
【系统提示：任务完成】
【获得奖励：灵石×200，经验×200】
【触发暴击：10 倍返利！】
【获得奖励：灵石×2000，经验×2000】
主角："还有这种好事？"
```

---

### 3.4 结尾（5 分钟）

**目标：** 500 字，设置悬念

**要点：**
- [ ] 事件收尾
- [ ] 角色反应
- [ ] 设置新悬念
- [ ] 引导下一章

**示例：**
```
任务堂恢复了平静。

李执事看着暴涨的修为，满意地点点头。

"这才刚刚开始..."

他望向远方，那里，更多的玩家正在涌入游戏。

而他不知道的是，
在宗门深处，
一双眼睛正默默注视着他...

（本章完）

【下一章预告：神秘长老的召见】
```

## 4️⃣ 自检阶段（10 分钟）

### 4.1 字数检查

- [ ] 统计本章字数
- [ ] 如果<3000 字，标记需要补充
- [ ] 如果>5000 字，考虑拆分

### 4.2 质量检查

使用 `chapter-checklist.md` 逐项检查：

- [ ] 12 项质量指标
- [ ] 爽点检查
- [ ] 角色检查
- [ ] 世界观检查
- [ ] 伏笔检查
- [ ] 文字检查

### 4.3 问题记录

记录发现的问题：
```
（例如：某处对话不自然、某处需要补充描写等）
```

---

## 5️⃣ 修改阶段（15 分钟）

### 5.1 优先修改

优先修改影响阅读的问题：
- [ ] 错别字
- [ ] 病句
- [ ] 逻辑漏洞
- [ ] 角色 OOC（性格不符）

### 5.2 次要修改

修改优化类问题：
- [ ] 补充描写
- [ ] 优化对话
- [ ] 调整节奏
- [ ] 增强爽点

### 5.3 最终通读

完整通读一遍：
- [ ] 是否流畅？
- [ ] 是否有卡顿？
- [ ] 是否有不满意的地方？

---

## 6️⃣ 发布阶段（5 分钟）

### 6.1 文件保存

- [ ] 保存到 `chapters/chapter-XXX.md`
- [ ] 文件命名：`chapter-001.md`、`chapter-002.md`...
- [ ] 确认保存成功

### 6.2 更新记录

- [ ] 更新 `daily-log.md`（创作日志）
- [ ] 更新 `foreshadowing-tracker.md`（如有伏笔）
- [ ] 更新 `outline.md`（如大纲有调整）

### 6.3 备份

- [ ] Git 提交（如已配置）
- [ ] 云端备份（如需要）

### 6.4 通知

- [ ] 通知老板审阅
- [ ] 准备收集反馈

## 📊 时间管理

### 标准时间分配

| 阶段 | 时间 | 占比 |
|------|------|------|
| 准备 | 5 分钟 | 5% |
| 大纲 | 10 分钟 | 10% |
| 创作 | 60 分钟 | 57% |
| 自检 | 10 分钟 | 10% |
| 修改 | 15 分钟 | 14% |
| 发布 | 5 分钟 | 5% |
| **总计** | **105 分钟** | **100%** |

### 灵活调整

根据实际情况调整：

**熟练后：**
- 创作阶段可缩短至 45 分钟
- 总时间可降至 90 分钟

**状态好时：**
- 可连续创作 2-3 章
- 但建议每章之间休息 15 分钟

**状态差时：**
- 不要勉强
- 可以只写大纲，明天再写

## 🎯 质量保障

### 每日目标

- [ ] 完成 1 章（3000-5000 字）
- [ ] 完成所有检查项
- [ ] 更新所有记录

### 每周回顾

每周末花 30 分钟回顾：
- [ ] 本周完成多少章？
- [ ] 质量是否稳定？
- [ ] 有哪些需要改进？
- [ ] 下周目标是什么？

### 每月总结

每月底花 1 小时总结：
- [ ] 本月完成多少章？
- [ ] 总字数多少？
- [ ] 读者反馈如何？
- [ ] 下月计划是什么？

## 💡 创作技巧

### 卡文时的解决方法

1. **回顾大纲** - 是不是偏离了主线？
2. **角色对话** - 让角色"自己说话"
3. **跳过难点** - 先写后面的，回头再补
4. **休息一下** - 起来走动，换个环境
5. **找人讨论** - 和老板讨论剧情走向

### 提升速度的技巧

1. **不要边写边改** - 先写完，再修改
2. **使用快捷键** - 熟悉编辑器快捷键
3. **准备素材库** - 常用描写提前准备
4. **定时创作** - 固定时间，形成习惯
5. **减少干扰** - 关闭手机通知

### 保持质量的方法

1. **坚持 SOP** - 不因为赶时间而跳过步骤
2. **定期回顾** - 每周/每月总结
3. **收集反馈** - 重视读者意见
4. **持续学习** - 看同类作品，学习技巧
5. **保持健康** - 适当休息，不熬夜
