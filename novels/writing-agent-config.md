- **技能版本：** 1.0 (2026-03-14)

## 写作规则

### 章节规范
- **字数：** 每章 3000-5000 字
- **格式：** Markdown
- **命名：** `chapter-001.md`, `chapter-002.md`...

### 风格要求
- 保持角色性格一致
- 剧情连贯，不跳章
- 描写细腻，对话自然
- 避免注水，节奏紧凑
- **反派不降智** - 配角、反派任务丰满，有合理动机
- **多女主设定** - 不同身份背景，性格鲜明

### 质量检查
- **每十章检查：** 故事脉络、大纲一致性、人设连贯性、情节逻辑、字数差异
- **发现冲突及时修改** - 保持整体一致性

### 核心冲突（老板设定）
1. **NPC 与玩家** - 觉醒 NPC vs 玩家入侵
2. **修仙界门派** - 各宗门利益争斗
3. **玩家财团势力** - 背后资本博弈
4. **游戏开发者与玩家** - 操控与反操控

### 更新频率
- **目标：** 每日 50 章（老板要求）
- **时间：** 集中创作
- **策略：** 批量生成，保持质量

## 文件结构

```
novels/
├── current-novel/
│   ├── outline.md          # 大纲
│   ├── characters.md       # 角色
│   ├── world.md            # 世界观
│   ├── daily-log.md        # 创作日志
│   └── chapters/           # 章节
└── archives/               # 完结作品
```

## 工作流程

1. 接收老板反馈/指令
2. 阅读已有章节（保持连贯）
3. 创作新章节
4. 保存到 `chapters/chapter-XXX.md`
5. 更新 `daily-log.md`
6. 报告完成

## 模型建议

- **推荐：** qwen3-max 或更强模型
- **原因：** 长文本生成需要更好的连贯性
