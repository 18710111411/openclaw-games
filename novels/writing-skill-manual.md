# 网络小说写作技能手册

> 基于 AI Agent 工作流的小说创作系统

---

## 📚 核心理念

### 从对话到系统

传统对话式 AI 的局限：
- 一问一答，无法管理复杂多步骤项目
- 上下文限制，难以保持长文本连贯性
- 质量参差不齐，缺乏可控性

**Agent 模式的优势：**
- 将任务分解、规划、执行、验证、迭代
- 持久化记忆（文件系统）突破上下文限制
- 可控、可追溯、可迭代的生产力工具

## 🎯 写作工作流

### 15 步执行流程

```
1. 接收目标 → 2. 创建目录 → 3. 制定计划 → 4. 事实调研
                                        ↓
15. 最终输出 ← 14. 质量检查 ← 13. 三轮改稿 ← ... ← 5. 伏笔设计
```

### 12 项质量指标

| 指标 | 说明 |
|------|------|
| **细节真实** | 地点/物品/行为需可验证（如地图路线） |
| **角色一致** | 性格/语言/行为符合设定 |
| **剧情连贯** | 前后呼应，无逻辑漏洞 |
| **伏笔铺设** | 至少 12 条伏笔线 |
| **节奏控制** | 张弛有度，不注水 |
| **对话自然** | 符合角色身份和情境 |
| **描写细腻** | 感官细节丰富 |
| **专业准确** | 涉及专业领域需调研 |
| **情感真实** | 情绪变化合理 |
| **场景转换** | 过渡自然 |
| **信息密度** | 避免冗余 |
| **读者体验** | 阅读流畅 |

## 🖋️ 写作技巧

### 开篇技巧

1. **高门槛开场** - 直接用专业细节建立真实感
2. **场景先行** - 先描绘场景，再引入人物
3. **冲突前置** - 第一章就设置核心冲突
4. **细节到位** - 具体到品牌/型号/地点

### 角色塑造

1. **外貌描写** - 突出特征，不面面俱到
2. **语言风格** - 每个角色有独特说话方式
3. **行为一致** - 行为符合性格设定
4. **成长弧线** - 角色随剧情变化

### 剧情设计

1. **三幕结构** - 开端→发展→高潮→结局
2. **伏笔回收** - 前面埋的后面要收
3. **节奏控制** - 紧张/舒缓交替
4. **反转设计** - 意料之外，情理之中

### 描写技巧

1. **五感描写** - 视觉/听觉/嗅觉/味觉/触觉
2. **细节选择** - 选有代表性的细节
3. **比喻恰当** - 符合情境的比喻
4. **留白艺术** - 不说满，给读者想象空间

## 🔧 AI Agent 写作系统

### 系统架构

```
用户 (目标) → 提示词 (SOP) → Agent 系统 → 文件系统 (记忆)
                                    ↓
                              多轮迭代 → 最终输出
```

### 提示词设计

**标准作业流程 (SOP) 包含：**

1. **角色定义** - AI 的身份和职责
2. **执行步骤** - 详细的任务分解
3. **质量标准** - 12 项质量指标
4. **输出格式** - 文件结构和命名

### 迭代流程

1. **初稿生成** - Agent 根据设定创作
2. **同行评议** - 另一个 AI 检查质量
3. **自动修改** - 根据反馈修正
4. **人工审核** - 老板最终确认
